OS第一次实验报告

思考题

Thinking 0.1

status不一样，因为提交了README.txt文件改变了git的状态

Thinking 0.2

Add the file 对应git中的add和commit
Stage the file对应git中的commit
commit 对应git中的push

Thinking 0.3

  1. 使用 git checkout – print.c
  2. 使用 git restore print.c
  3. 使用 git rm –cached print.c

Thinking 0.4

发现在第三次尝试无法成功，无法再从旧版本返回新版本

Thinking 0.5

第一条指令在屏幕上显示出了first
第二条指令使output.txt的内容变为了second
第三条指令使output.txt的内容变为了third
第四条指令使output.txt的内容变为了thirdforth

Thinking 0.6

对于echo echo Shell Start和echo `echo Shell Start`，前者会输出echo Shell Start，后者会输出Shell Start
对于 echo echo $c>file1和echo `echo $c>file1`，前者会输出echo $c到file1，后者会输出$c到file1中

难点分析

  1. 最大的难点来自于记不住各个指令的格式、功能和使用场景
  2. 其次，难点来自于无法在本地编写代码，机房提供的电脑并未安装git，导致我只能在web终端上进行代码的编写，由于对vim的操作不够熟悉、缺少代码补全功能等原因，导致我实验效率极低。
  3. 还有难点来自于对gcc具体操作了解不够深入，对C代码如何编译成可执行文件的过程缺乏明确的认识。

实验体会

这次lab0实验使我难得地回忆起了去年大一上学期做程序设计上机的时候在机房汗流浃背的感觉。但同时也让我意识到了自己的不足，我会争取在接下来的实验中努力追赶上来，以在接下来的实验中取得好成绩。
